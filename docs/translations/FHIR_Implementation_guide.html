FHIR実装ガイドの紹介

FHIRはプラットフォーム仕様です。多くの医療データ交換の問題を解決するために使用できる機能の一般的なデータセットについて説明します。FHIRは、さまざまなアーキテクチャと情報交換パターンを使用する多くの管轄区域であらゆる種類の問題に使用されるため、FHIR仕様は汎用性があり、問題を解決する方法は多数備えています。このため、実装者は方法を選択する必要があります。特定のソリューションで相互運用可能にするには、同じ実装を選択する必要があります。

FHIR実装ガイド（IG; Implementation Guide）は、特定のコンテキストでFHIRがどのように使用されるかを説明するものです。多くのIGは、次の3つの分類のいずれかに適合します。

  1. 司法管轄区：司法管轄区でFHIRを使用するためのベースのルールを説明します（通常は各国特有のルールセット）
  2. アプリケーションソリューション：FHIRを使用して特定の問題（EHRアプリアクセスなど）を解決する方法を説明します
  3. ドメインガイド：FHIRで特定のコンテンツを表す適切な方法を説明します（例：乳がんの診断プロセスを適切に表す方法）

IGの中にはカテゴリを超えたり、これらの主要なカテゴリのいずれにも該当しない場合があることに注意してください。

実装ガイドの見つけ方

FHIR財団は、IGの中央レジストリを維持しています。公開されているIGはすべて登録することができます（手順を参照）ので、IGを書いたら登録することをお勧めします。

FHIR財団は、きめ細かい情報セットを含む一般的なFHIRリポジトリも維持しています。このレジストリには、fhir.orgおよびhl7.orgによって公開された実装ガイドにある適合制作物（拡張、プロファイル、値セットなど）が含まれますが、これらに限定されません。実装ガイドを書く際には一般的なレジストリを最新の状態に保つことをお勧めします。レジストリのコンテンツがIGアップロードプロセスを通じて登録されている場合、レジストリはResourceが公開されているIGへのリンクを提供します。

さらに、実装ガイドはGoogleなどを使用して見つけることができます。

実装ガイドの使い方

実装ガイドの主な用途は、オンラインで読むことです。

ほとんどのガイドにはダウンロード機能があり、オフラインでもローカルで使用できます。しかし、ガイドの中で相互に参照しているところは、ローカルダウンロードコピーにはリンクされず、Webロケーションを参照することに注意してください。

さらに、ガイドには、コンテンツの一部として（ FHIR実装ガイド公開要件を参照）、検証およびコード生成ツールでの使用に適したファイルのセットが含まれています。FHIR IGをサポートするツールは、IGの場所（ルートURL）（ローカルコピーまたはWeb）が提供されると、このツールの場所を認識します。

FHIR IGにはNPMパッケージの説明があるため、NPMを使用してIG依存関係を管理できます。

FHIR実装ガイドの書き方

実装ガイドを作成するための一般的なワークフローは次のとおりです。

  • はじめてみよう
    • 実装ガイド（IG）、その範囲、および正規のURLを記載してください
      • HL7ガイドの場合、正規URLのルートはhttp://hl7.orgになります。他のガイドであれば、http://fhir.org 、 http://ihe.netなどとなるかもしれません。
    • IGのテンプレートを選択します（ FHIR IG Publisher templatesを参照）
    • コンテンツを記載していない初期ImplementationGuideリソースを作成します
    • IGのバージョン管理を設定します
    • IG Publisherを使用してIGを構築する
      • githubリポジトリが公開されている場合、自動公開を設定して、ガイドの最新バージョンが常にbuild.fhir.orgから見つかるようにしておきます。
  • コンテンツの開発
    • 実装について記述された適合リソース（以下を参照）を追加します
    • 解説を書きます（好みに応じてhtml、xhtml、またはマークダウン）
    • 例、画像などを追加して、実装ガイドに記入します
    • ローカルビルドを実行して、すべてのコンテンツがすべて正常であることを確認します
      • CIビルドサイトに公開することをコミットします（セットアップの場合、全員が現在の作業を確認できるようにします）
  • 公開（完了後）
    • マイルストーンビルドは、正規の場所に発行する必要があります（生成されたHTMLファイルのセットをコピーします）
    • 正規の場所のレイアウトは通常、 Standard FHIR Publishing Layout Patternに従う必要があります
    • hl7.orgまたはfhir.orgで公開されるコンテンツは、FHIR IG Publihsherを介して生成される必要があります

通常、IGは、公式コンセンサスビルドで下書きされてから公開されるため、開発と公開を何度も繰り返します。

一般的な実装ガイドのコンテンツの種類

通常、実装ガイドには次の種類のコンテンツが含まれます。

  • Resource
    • ProfileとExtensionの定義-ガイドの対象となるResourceのコンテンツモデルの定義
    • 値セットとコードシステム-実装ガイドの用語定義をサポート
    • 機能定義と検索パラメーターと共に機能ステートメント-FHIR APIの使用方法を定義します
    • テストケース（テストスクリプト）-実装がパスする、またはパスする必要がある実際のテストのセット
    • ExampleScenaro-IGのストーリーボードの正式な定義（テストスクリプトまたは他のテストフレームワークを駆動する場合があります）
    • ConceptMapおよびStructureMap-ガイドで説明されているFHIRコンテンツとの間のマップおよび変換
    • 論理モデル- 実装ガイドの論理モデルのドキュメントを参照
    • 例-実装ガイドをさらに説明するあらゆる種類のResource
  • 解説：
    • プロセス情報： バージョンとライセンス、発行者/所有者、問題の報告/議論への参加
    • セキュリティ情報：セキュリティの仕組み（アプリケーションソリューションタイプガイド用）
    • ドメイン分析/背景情報
    • UML図、BPMNフロー図、その他の種類の図

これらのコンテンツはすべてオプションであり、IGの実際のコンテンツと解決される問題に応じて適切であることに注意してください。IGを書くときには、開発中にIGのベストプラクティスガイドラインに留意する必要があります

IGソースの例をご覧になりたい場合は、次のいずれかの場所をご覧ください。

  • FHIR GitHubリポジトリ、/guides /* の中にある2で終わるディレクトリにはさまざまなスタイルの実装ガイドがあります。
  • また、ツールのテストに使用される/tests/igがあります

FHIR実装ガイドを書くときのツールサポート

FHIR IGを公開するための主なツールチェーン：

  • FHIR IG Publishingツール

  • プロファイルの編集/拡張機能の定義/論理モデル：Forge
  • 実装ガイドの編集：Forge（制限付き）

  • シナリオ例の編集：
  • アンケートの編集：
