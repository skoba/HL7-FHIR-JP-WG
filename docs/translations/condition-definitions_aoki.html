9.2.6 Resource Condition - Detailed Descriptions

患者ケアワークグループ成熟度 ：3 試用セキュリティカテゴリ ：患者 コンパートメント ： 出会い 、 患者 、 開業医 、 RelatedPerson

Detailed Descriptions for the elements in the Condition resource.

Condition
Element IdCondition
Definition

懸念レベルにまで上昇した臨床状態(conditon)、問題(problem)、診断(diagnosis)、またはその他のイベント(event)、状況(situation)、問題(issue)、または臨床概念(clinical concept)。

Cardinality0 ..*
TypeDomainResource
Invariants
この要素で定義する
con-3Guideline validationStatusが”entered-in-error"ではなく、categoryがproblem-list-itemの場合、Condition.clinicalStatusが存在する必要があります。(SHALL)clinicStatus.exists（）またはvalidationStatus = 'entered-in-error'またはcategory.select（$ this = 'problem-list-item').empty（）
これは、（唯一の）ベストプラクティスガイドラインです：

多くのシステムは、時間経過とともに管理されるproblem-list-itemsのclinicalStatusが評価されることを期待しますが、特定の時点でのencounter-diagnosisのためにclinicalStatusは必要ない場合があります。

con-4Rule conditionが緩和された場合、clinicalStatusは非アクティブ(inactive)、治癒(resolved)、または寛解（remission）でなければなりませんabatement.empty（）またはclinicStatus.coding.where（system = 'http：//terminology.hl7.org/CodeSystem/condition-clinical'および（code = 'resolved'またはcode = 'remission'またはcode = 'inactive' ））.exists（）
con-5Rule verification Status が "entered-in-error" の場合、Condition.clinicalStatusは存在してはなりません。(SHALL NOT)validationStatus.coding.where（system = 'http：//terminology.hl7.org/CodeSystem/condition-ver-status' and code = 'entered-in-error'）.empty（）またはclinicStatus.empty（）
Condition.identifier
Element IdCondition.identifier
Definition

実行者または他のシステムによってこのconditionに割り当てられたビジネス識別子。resourceが更新されたり、サーバー間で伝送されたりする間、変化せず一意である。

Noteこれはビジネス識別子であり、リソース識別子ではありません。（ 説明を参照）
Cardinality0 ..*
TypeIdentifier
Requirements

Condition.identifierの一意の識別を可能とする。これはさまざまなシステムで認識されており、サーバー間で一貫性を保つことができる。

SummaryTrue
Comments

これはビジネス識別子であり、リソース識別子ではありません（ 説明を参照）。識別子は単一のリソースインスタンスにのみ表示されることがベストプラクティスですが、ビジネスプラクティスでは、同じ識別子を持つ複数のリソースインスタンスが存在するように指定される場合があります。たとえば、複数のPatientおよびPersonリソースインスタンスが同じ社会保険番号を共有する場合があります。

Condition.clinicalStatus
Element IdCondition.clinicalStatus
Definition

状態(condition)の臨床状態(clinical status)。

Cardinality0..1
Terminology BindingCondition Clinical Status Codes (Required)
TypeCodeableConcept
Is Modifiertrue（理由：この要素は修飾子(Modifier)としてラベル付けされています。ステータスにアクティブではないことを示すコードがステータスに含まれている。）
SummaryTrue
Comments

ClinicalStatusには何らかの臨床判断(clinical judgment)が関係しているため、データタイプはCodeableConceptです。そのため、もとのFHIR値セットがカバーするよりも具体的な値セットが必要になる場合があります。たとえば、SNOMEDコーディングを使えば、さらに詳しくできるでしょう。

Condition.verificationStatus
Element IdCondition.verificationStatus
Definition

状態(condition)の臨床状態(clinical status)をサポートする証拠となるステータス(verification status)。

Cardinality0..1
Terminology BindingConditionVerificationStatus （ 必須 ）
TypeCodeableConcept
Is Modifiertrue（理由：この要素は修飾子としてラベル付けされます。このステータスには、Condition.verificationStatusが現在無効であると示すための、refuteコードや"entered-in-error"が含まれているためです。）
SummaryTrue
Comments

validationStatusは必要ありません。たとえば、救急病棟において患者が腹痛がある場合、verification statusになる可能性はほとんどありません。verificationStatusには何らかの臨床判断(clinical judgment)が関係しているため、データタイプはCodeableConceptです。そのため、もとのFHIR値セットがカバーするよりも具体的な値セットが必要になる場合があります。たとえば、SNOMEDコーディングを使えば、さらに詳しくできるでしょう。

Condition.encounter
Element IdCondition.encounter
Definition

このConditionが作成された、またはこの記録の作成が緊密に関連付けられたencounter(ヘルスケアサービスを提供したり、患者の健康状態を評価したりするための、患者とヘルスケア提供者との相互作用やその機会のこと)。

Cardinality0..1

Condition.abatement [x]
Element IdCondition.abatement [x]
Definition

Conditionが治癒(resolved)または寛解(remission)した日付or推定日付。これは、寛解(remission)または治癒(resolution)に関連する用語の多くが過剰な意味を持って使われすぎているため「緩和」(abatement)と呼ばれます。ーConditionは実際には治癒(resolved)しませんが、緩和(abate)することができます。

Cardinality0..1
TypedateTime|Age|Period|Range|string
Note使用方法の詳細については、データ型の選択を参照してください[x]
Comments

多くの場合、区別が明確ではないため、resolutionとremissionの明確な区別はありません。通常、年齢は、patientのconditionが緩和した年齢を報告するときに使用されます。abatement elementがない場合、conditionがresolveしたかremissionだったか不明です。Applicationとuserは通常、conditionがまだ有効であると想定する必要があります。(should)abatementStringが存在する場合、conditionが緩和されたことを意味します。

Invariants
この要素に影響する
con-4Rule conditionが緩和された場合、clinicalStatusはinactive、resolved、またはremissionでなければなりませんabatement.empty（）またはclinicStatus.coding.where（system = 'http：//terminology.hl7.org/CodeSystem/condition-clinical'および（code = 'resolved'またはcode = 'remission'またはcode = 'inactive' ））.exists（）
