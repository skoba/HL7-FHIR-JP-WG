9.1.6 リソースアレルギー不耐症-詳細な説明

AllergyIntoleranceリソースの要素の詳細な説明。

アレルギー不耐症
Element IDAllergyIntolerance
Definition

患者に特有な、物質への曝露に関連する有害または望ましくない生理学的反応のリスク。

Cardinality0 ..*
TypeDomainResource
Requirements

患者に対する、特定の物質や類似物質への暴露に対しての副作用の傾向や、潜在的なリスクについての臨床的評価を記録するためのもの。

Alternate Namesアレルギー;不耐性;副作用
Comments

物質には以下のものが含まれる。患者に適切な用量で正しく投与された薬剤、食物;植物または動物に由来する物質;虫刺されからの毒。

Invariants
この要素で定義されるルール
ait-1ルール validationStatusが"entered-in-error"でない場合、AllergyIntolerance.clinicalStatusが存在する必要があります。validationStatus = 'entered-in-error'またはclinicStatus.exists（）
ait-2ルール verificationStatusが"entered-in-error"になっている場合は、AllergyIntolerance.clinicalStatusは存在しないものとします。validationStatus != 'entered-in-error'またはclinicStatus.empty（）

Cardinality0 ..*

AllergyIntoleranceの一意の識別を可能とする。これは、さまざまなシステムで認識されており、サーバー間で一貫性を保つことができる。

Summarytrue

これはビジネス識別子であり、リソース識別子ではありません（ 説明を参照）。識別子は単一のリソースインスタンスにのみ表示されることがベストプラクティスですが、ビジネスプラクティスでは、同じ識別子を持つ複数のリソースインスタンスが存在するように指定されることがあります。たとえば、複数のPatientおよびPersonリソースインスタンスが同じ社会保険番号を共有する場合などがあります。

AllergyIntolerance.type
Element IDAllergyIntolerance.type
Definition

副作用リスクの原因にある生理学的メカニズムの種類。

Cardinality0..1
Terminology BindingAllergyIntoleranceType （ 必須 ）
TypeCode
Alternate Namesカテゴリー;クラス
Summarytrue
Comments

アレルギー（通常は免疫介在性）反応は、歴史的には、重大な将来のリスクへつながる潜在的な指標と見なされてきました。現代では、以前は免疫介在性であると考えられていたいくつかの反応は、実際には非免疫性であるが、場合によっては依然として生命を脅かすリスクをもたらす可能性があることが示唆されています。多くの臨床医が特定の反応のメカニズムを区別しないかもしれないことが指摘されています。多くの場合、「アレルギー」という用語はかなり一般的に使用され、「不耐性」と重複することがあります。実際には、これら2つの概念の境界は明確に定義または理解されていないかもしれません。にもかかわらず、多くのレガシーシステムがこの属性を記録しているので、このデータ要素が存在します。免疫学的検査により、反応の基盤と原因物質を裏付けるエビデンスを得られるかもしれないが、特定の物質に対する感度または特異的が100％な検査はありません。よくあることですが、もし反応がアレルギーによるものか不耐性によるものかが不明な場合は、リソースからtype要素を省略してください。

AllergyIntolerance.category
Element IDAllergyIntolerance.category
Definition

特定された原因物質のカテゴリ。

Cardinality0 ..*
Terminology BindingAllergyIntoleranceCategory （ 必須 ）
TypeCode
Alternate Namesカテゴリー;タイプ;反応タイプ;クラス
Summarytrue
Comments

このデータ要素は、現在一部の臨床システムで取得されているため、含まれています。このデータは、コーディングシステムが使用されている"Substance"から取得することができ、取得できる場合、事実上冗長なデータです。カテゴリーを検索するときは、カテゴリーなしのAllergyIntoleranceリソースの影響を考慮してください。たとえば、カテゴリ=薬で検索する場合、カテゴリの値が設定されていない薬物へのアレルギーは返されません。カテゴリを持たないアレルギーを取得するには、:missing修飾子を使用してカテゴリを検索します。詳細については、 searchを参照してください。また、カテゴリは送信者の主観的なものになる可能性があります、注意して使用してください。

AllergyIntolerance.criticality
Element IDAllergyIntolerance.criticality
Definition

特定されたアレルギー物質に対する反応の、潜在的な臨床的危害または深刻さの推定。

Cardinality0..1
Terminology BindingAllergyIntoleranceCriticality （ 必須 ）
TypeCode
Alternate Names重大度(Severity);深刻さ;禁忌;リスク
Summarytrue
Comments

有害反応の傾向のデフォルトの"criticality"は「低リスク」であるべきです。これは、物質への意図的または自発的暴露に対しての、少なくとも相対的禁忌のレベルを示します。臨床医がアナフィラキシーなどのより深刻な、または生命を脅かす可能性のある反応の傾向を特定した場合、「高リスク」にフラグが立てられ、物質への意図的または自発的暴露に対する絶対的な禁忌を意味します。この要素が欠落している場合、"Criticality"は不明です（ただし、他の所で表現されているかもしれません）。"condition"レベルで"Severity"をキャプチャするシステムでは、実際には"criticality"の概念を表していますし、"reaction"レベルで文書化された"Severity"は真の反応の"Severity"を表しています。"condition"の"criticality"と"reaction"の"severity"の両方をキャプチャしている既存のシステムでは、"Severity"という用語を使用して両方を表す場合があります。"Severity"は状況に依存しないのに対し、"Criticality"は将来可能性として最悪になり得ます（つまり、状況に依存しない）。

AllergyIntolerance.code
Element IDAllergyIntolerance.code
Definition

アレルギーまたは不寛容に関する表現のコード（肯定的または否定/除外、などの表現）。これは、以下のいずれかである。 ・有害反応のリスクの原因と考えられた物質または医薬品のコード（例：「ラテックス」） ・アレルギーと不耐性の状態についてのコード（例、「ラテックスアレルギー」） ・特定の物質やクラスが原因ではない時の、否定・除外を表すコード（例：「ラテックスアレルギーなし」） ・一般的な否定文、またはカテゴリに対する否定文を表す、否定・除外を表すコード（例：「既知のアレルギーなし」、「既知の薬物アレルギーなし」）注：特定の一回の反応に対する物質は、リスクの原因として特定された物質とは異なる場合があるが、それらは一貫している必要がある。たとえば、具体的なアレルギー物質（たとえば、ある薬のブランド）と、その物質を含む複合製品などがある。「code」のみを処理し、「reaction.substance」を無視することが、臨床上安全であることを保証しなければならない。受信側のシステムがAllergyIntolerance.reaction.substanceで詳細化された情報が、AllergyIntolerance.codeの表現する物質のスコープ内にあることを確認できない場合、受信側システムはAllergyIntolerance.reaction.substanceの方を無視するべきです。

Cardinality0..1
Terminology Bindingアレルギー不耐性物質/製品、状態および否定コード （ 例 ）
TypeCodeableConcept
Alternate Namesコード
Summarytrue
Comments

この要素は、可能であれば、用語集を利用して設定することを強くお勧めします。たとえば、用語集には、RxNorm、SNOMED CT、DM + D、NDFRT、ICD-9、IDC-10、UNII、ATCなどがあります。プレーンテキストは、使用可能な適切な用語がない場合にのみ使用してください。追加の詳細をテキストで指定できます。

「コード」要素に物質または製品コードが指定されている場合、「デフォルト」では、これが物質または製品に対するアレルギーまたは不耐性の（「タイプ」要素が存在する場合は、その要素の値による）の肯定的なステートメントであることを意味する。対応するSNOMED CTアレルギーモデルでは、指定された物質/製品が「原因物質」関係のターゲット（宛先）です。

「substanceExposureRisk」拡張機能は、肯定的または否定的なアレルギー・不耐性のstatementを作成するための、「コード」要素の構造化されたより柔軟な代替手段として利用できます。この拡張は、コード化された物質/製品に関する「既知のアレルギーなし」（または「有害反応のリスクなし」）statementを作成する機能を提供します（その物質/製品の事前調整された「xへのアレルギーなし」の概念が存在しないケースを含む）。'substanceExposureRisk'拡張が存在する場合、AllergyIntolerance.codeエレメントは省略されるべきである。

AllergyIntolerance.note
Element IDAllergyIntolerance.note
Definition

他のフィールドでは記載されていない、副作用の傾向に関する追加の記述。

Cardinality0 ..*
TypeAnnotation
Comments

例：「高リスク」の重大性にフラグを立てる理由を含める。集中治療室内や、ステロイドカバー下など、将来的な物質の暴露または投与に関する指示。メモは、一般的な状態としてのアレルギーまたは不耐性に関連するものであるべきであり、特定のエピソードに関連するものを記述すべきではない。特定のエピソードについてのメモ・説明には、AllergyIntolerance.event.descriptionとAllergyIntolerance.event.notesを使用してください。

AllergyIntolerance.reaction.substance
Element IDAllergyIntolerance.reaction.substance
Definition

有害事象イベントの原因と考えられた特定の物質（または医薬品）の識別情報。注：特定の一回の反応に対する物質は、リスクの原因として特定された物質とは異なる場合があるが、それらは一貫している必要がある。たとえば、具体的なアレルギー物質（たとえば、ある薬のブランド）と、その物質を含む複合製品などがあります。「code」のみを処理し、「reaction.substance」を無視することが、臨床上安全であることを保証しなければならない。受信側のシステムがAllergyIntolerance.reaction.substanceで詳細化された情報が、AllergyIntolerance.codeの表現する物質のセマンティックスコープ内にあることを確認できない場合、受信側システムはAllergyIntolerance.reaction.substanceの方を無視するべきです。

Cardinality0..1
Terminology Binding物質コード （ 例 ）
TypeCodeableConcept
Comments

可能な限り、特定の物質（または医薬品）を、意思決定支援システムをトリガーできる用語でコーディングするべきです。'.code'要素は、特定の物質または医薬品だけでなく、物質のグループまたはクラスの使用を許しています。'code'で指定されたものが、物質のクラスに対するアレルギーまたは不耐性の場合（「ペニシリン系薬物」など）、「reaction.substance」要素を利用して、反応を引き起こしたと特定された物質をコーディングできます（例えば、 「アモキシシリン」）。「code」および「reaction.substance」要素の値の重複は、特定の物質が「code」に記録されている場合に許されます。
